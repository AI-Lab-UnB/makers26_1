# Resumo Técnico: Git & GitHub Crash Course 2025

## 1. Arquitetura e Conceitos Fundamentais
A base do desenvolvimento moderno reside na distinção entre a ferramenta de versionamento e a plataforma de colaboração.

* **Git (VCS Distribuído):** Diferente de sistemas centralizados, o Git permite que cada desenvolvedor possua uma cópia integral do histórico do projeto. Isso garante alta performance e redundância.
* **GitHub:** É uma plataforma de hospedagem de código-fonte baseada na nuvem, que usa o Git para o controle da versão.
* **Workflow do Git:**
    1.  **Working Directory:** Edição física dos arquivos.
    2.  **Staging Area (Index):** Ambiente temporário para seleção de mudanças.
    3.  **Local Repository:** Armazenamento definitivo dos commits.
    4.  **Remote Repository:** Armazenamento do código na nuvem para compartilhamento com outros usuários. 

## 2. Configuração e Operações Locais
Comandos essenciais:
 
* **Identidade Global:** Configuração de `user.name` e `user.email` para atribuição de autoria. **Exemplo:** `git config --global user.name <Nome do usuário>`
* **Inicialização:** O comando `git init` cria o diretório oculto `.git`, o "cérebro" do repositório.
* **Gerenciamento de Estado:**
    * `git add .`: Prepara modificações para o commit.
    * `git commit -m "mensagem"`: Registra o ponto de restauração.
    * `git status` e `git log`: Ferramentas de diagnóstico e histórico.

## 3. Segurança e Gestão de Arquivos (.gitignore)
O uso do arquivo **`.gitignore`** é uma prática de segurança indispensável. Ele instrui o Git a ignorar deliberadamente:
* Arquivos de sistema e dependências (ex: `node_modules`).
* **Dados Sensíveis:** Chaves de API, credenciais e arquivos `.env`, evitando o vazamento de segredos em repositórios remotos.

## 4. Branching e Fluxo de Pull Requests
A técnica de ramificação permite o desenvolvimento isolado e seguro de novas funcionalidades.

* **Isolamento de Código:** Com `git checkout -b`, cria-se uma linha do tempo paralela para experimentos ou correções sem afetar a branch principal (`main`).
* **Pull Requests (PRs):** O mecanismo de governança do GitHub. Permite a revisão por pares, discussão e testes antes da integração final (*merge*).
