1-Resumo Técnico:

O que é Git: Sistema de controle de versão distribuído. Imagine como uma "máquina do tempo" para o seu código.

Fluxo de Trabalho (Workflow):

    Working Directory: Onde você altera seus arquivos.

    Staging Area: Onde você prepara os arquivos para a foto (commit) usando git add.

    Local Repository: Onde as mudanças são salvas sem ir para a nuvem, usando git commit.

    Remote Repository: O servidor (nesse caso o github) onde você envia o código com git push.

Comandos Essenciais:

    git init: Inicializa o repositório.

    git clone: Clona um repositório existente.

    git pull: Traz as alterações feitas em um repositório para a máquina local do usuário.

    git branch: Cria ramificações para testar novas funcionalidades sem alterar o código principal.

Markdown: É a linguagem usada no arquivo README.md, muito usada para textos importantes, como descrição de arquivos e útil para a formatação.
