# Relatório de Atividades e Reflexão

## O que é o Git e Github?

### **Git** 
- O git é uma ferramenta criada por Linus Torvalds, inicialmente foi desenvolvida com o objetivo de ajudar no controle de versionamento do Kernel do futuro sistema operacional Linux. Atualmente o Git é o maior sistema de controle versão distribuido do mundo

- **Fluxo de trabalho do Git / Git Workflow**

1. **Working Directory:** Local onde o usuário edita e cria arquivos.
2. **Staging Area:** Local onde os arquivos são adicionados para atualizarem o repositório local sendo commitados.
3. **Local Repository:** Local onde o Git atualiza o repositório local do usuário, garantindo as mudanças feitas do programa.
4. **Remote Repository:** Repositório no Github onde o código é enviado, ficando vísivel para o time.

Atualmente o Git é utilizado como a maior ferramenta do mundo para admnistrar projetos e coordenar o fluxo de trabalho.

- **Outras Utilidades importantes do _Git_.**

1. **Branching/Branches :** Criação de ramificações do código, permitindo o desenvolvimento separado e organizado de features diferentes, facilitando a organização entre o time.

2. **Pull Request :** Solicitar um pedido de mesclagem à branch principal a partir de outra branch, isso garante que um código seja revisado e testado antes de ser aplicado no sistema funcional.

3. **Uso do arquivo _.gitignore_ :** Arquivo responsável por tornar arquivos _invisíveis_ ao Git, garantindo que arquivos com senhas importantes ou informações sensíveis sejam enviados ao repositório público, garantindo segurança.

- **Comandos Relevantes do Git:**

  `git init :` **Inicializar um repositório local na sua máquina**

  `git status :` **Checa o status atual do seu fluxo de trabalho dentro da branch atual**

  `git log :` **Mostra os registros dos commits realizados na branch atual**

  `git clone <link_repositorio> :` **Clona o repositório do Github para sua máquina local**

  `git remote add origin <link_repositorio> :`**Cria um vínculo de um repositório local à um repositório online**

  `git pull origin <nome_branch> :` **Atualiza os arquivos/versão do seu repositório local com a versão mais atual da branch escolhida**

  `git push origin <nome_branch> :` **Atualiza o repositório online com os arquivos criados/alterados localmente na branch escolhida**

  `git add <nome_arquivo> ou <.> :` **Coloca os arquivos escolhidos dentro da _Staging area_.**

  `git commit -m "<mensagem>" :` **Coloca os arquivos escolhidos dentro do _Local Repository_ com uma descrição.**


### **Github**
- O **Github** é um site que armazena **repositórios** virtuais públicos e privados para auxiliar equipes de desenvolvedores a criar projetos de software de forma organizada. Garantindo consistência do grupo, padrões de desenvolvimento e regras de negócio implementadas diretamente no ambiente de desenvolvimento.
