

# Git & GitHub — Resumo Técnico

- **Base:** Git & GitHub Crash Course 2025 (Traversy Media)
- **Nível:** Iniciante
- **Objetivo:** Fluxo prático de versionamento e deploy

## Visão Geral Arquitetural

O Git é um **Sistema de Controle de Versão (VCS) distribuído**. O histórico é armazenado localmente na pasta oculta `.git`.

### As Três Áreas do Git
1. **Working Directory:** Onde você edita seus arquivos.
2. **Staging Area:** Área de preparo antes do salvamento.
3. **Local Repository:** Onde o Git armazena os snapshots (versões) do projeto.

## Fase 1 — Configuração Inicial

Configuração de identidade (necessária apenas uma vez):

```bash
git config --global user.name "Seu Nome"
git config --global user.email "seu-email@exemplo.com"
git config --global init.defaultBranch main
```

### Autenticação SSH
Para conectar ao GitHub com segurança:
1. Gerar chave: `ssh-keygen -t rsa -b 4096`
2. Iniciar agente: `eval "$(ssh-agent -s)"`
3. Adicionar chave: `ssh-add ~/.ssh/id_rsa`
4. Copiar conteúdo do arquivo `.pub` e colar no GitHub (Settings -> SSH keys).

## Fase 2 — Ciclo Básico

### Começando um projeto
* **Novo:** `git init` (inicializa a pasta local).
* **Existente:** `git clone <url_do_github>` (baixa um projeto pronto).

### Fluxo de Trabalho Diário
1.  `git status`: Verifica quais arquivos foram alterados ou não rastreados.
2.  `git add <arquivo>` ou `git add .`: Move as mudanças para a Staging Area.
3.  `git commit -m "mensagem explicativa"`: Salva as alterações no repositório local.
4.  `git log`: Exibe a lista de commits realizados.

## Fase 3 — Proteção com .gitignore

Arquivos listados no `.gitignore` não são enviados para o GitHub. Essencial para segurança (esconder chaves de API e arquivos `.env`).

**Exemplo de conteúdo:**
```text
.env
node_modules/
```

## Fase 4 — GitHub e Repositório Remoto

### Sincronização
1. **Vincular repositório:** `git remote add origin <url_ssh_do_repositorio>`
2. **Enviar (Push):** `git push -u origin main` (o `-u` vincula as branches para os próximos envios).
3. **Puxar (Pull):** `git pull` (baixa as novidades enviadas por outros colaboradores).

## Fase 5 — Branches e Colaboração

Branches permitem trabalhar em novas funcionalidades sem quebrar o código principal (`main`).

1.  **Criar e entrar na branch:** `git checkout -b feature/login`
2.  **Ver branches existentes:** `git branch`
3.  **Trocar de branch:** `git checkout main`
4.  **Integrar mudanças:** Via **Pull Request (PR)** na interface do GitHub.
5.  **Excluir branch local:** `git branch -d feature/login`

## Fase 6 — CI/CD com Vercel

Integração Contínua e Deploy Contínuo:
* Conecte sua conta do GitHub à **Vercel**.
* Importe o repositório.
* **Fluxo automático:** Sempre que você der um `git push` para a branch `main`, a Vercel detecta e atualiza o site automaticamente em segundos.

## Resumo de Comandos Rápidos

| Comando | Função |
| :--- | :--- |
| `git init` | Inicia repositório local |
| `git status` | Vê o estado dos arquivos |
| `git add .` | Prepara todos os arquivos |
| `git commit -m ""` | Salva a versão localmente |
| `git push` | Envia para o GitHub |
| `git pull` | Baixa do GitHub |
| `git checkout -b <nome>` | Cria nova branch |

# O que é Desenvolvimento Profissional?
 
Desenvolvimento profissional vai muito além de acumular diplomas ou dominar a ferramenta do momento. É um compromisso contínuo com a própria evolução numa área que muda antes mesmo de você terminar de aprender o que começou ontem.
 
Esse desenvolvimento se apoia em três dimensões. A primeira é o **aprendizado aplicado**: o conhecimento que realmente fica é o que você conquistou na prática — errando numa implementação, resolvendo um bug impossível ou construindo um projeto que ninguém pediu, mas que você precisava fazer para entender. A segunda é a **autocrítica estratégica**: crescer de forma sustentável exige pausas deliberadas para avaliar o que funcionou, identificar gargalos e reformular o que não está servindo — saber fazer essa leitura da própria trajetória é uma das habilidades mais raras da área. A terceira é a **construção em blocos lógicos**: habilidades técnicas aprofundadas combinadas com comunicação, colaboração e o hábito de documentar o próprio progresso formam uma base sólida e legível tanto para você quanto para o mercado.
 
No fim, desenvolvimento profissional é a soma de escolhas pequenas e consistentes feitas ao longo do tempo. Não existe atalho, mas existe método — e reconhecer que você já está construindo esse método é, por si só, um sinal claro de maturidade profissional.
