# Resumo Técnico: Git & GitHub Crash Course

## Git
### O que é o Git?
É um **sistema de controle de versão (VCS) distribuído** que ajuda desenvolvedores a rastrear seus códigos-base, colaborar com outras pessoas e gerenciar múltiplas versões de um projeto.

O Git é **descentralizado**, o que significa que você não precisa de um servidor central. Em vez disso, cada desenvolvedor possui uma cópia completa do repositório em sua própria máquina.

### Principais Características do Git
* **Distributed System**: É descentralizado, ou seja, não precisa de um sistema central.
* **Version Tracking**: Mantém o controle das alterações feitas em sua base de código.
* **Collaboration**: Vários desenvolvedores podem trabalhar no mesmo projeto simultaneamente.
* **Branching**: Permite isolar o desenvolvimento de novas funções ou correções. Garante que a branch principal (`main`) permaneça estável enquanto o trabalho está em progresso.
* **Merging**: O processo de integrar as mudanças de uma branch secundária de volta para a branch principal após a conclusão e revisão.
* **Remote Repositories**: Como o GitHub.
* **Extensive Tooling**: O Git é compatível com várias ferramentas, incluindo IDEs (como VS Code e IntelliJ).
* **Staging Area**: Área onde você coloca arquivos selecionados antes de fazer o commit para o repositório local.
* **Speed**: O Git é extremamente rápido e eficiente.
* **Open-Source & Free**: Código aberto e gratuito.

## GitHub
### O que é o GitHub?
Plataforma baseada na web usada para controle de versão e colaboração. Ele hospeda repositórios e fornece uma interface para gerenciar seu código e realizar diversas outras tarefas.

* **Recursos**: Oferece ferramentas como *bug tracking*, *feature requests*, gestão de tarefas e *wikis*.
* **Diferença Crucial**: Git é a ferramenta (VCS) e o GitHub é o serviço que hospeda os repositórios na nuvem.
* **Alternativas**: GitLab e BitBucket.

**configurando Chaves SSH** 
As chaves SSH servem como uma "assinatura digital" segura que permite que você se conecte ao GitHub sem precisar digitar sua senha a cada comando.

**1. Gerar uma nova chave SSH**
No terminal, execute o comando abaixo substituindo pelo seu e-mail do GitHub:
```bash
ssh-keygen -t ed25519 -C "seu.email@exemplo.com"
Dica: Quando perguntar o local de salvamento, apenas aperte Enter.
```

**2. Adicionar a chave ao agente SSH**

Liga a chave no sistema:
```bash
eval "$(ssh-agent -s)"
ssh-add ~/.ssh/id_ed25519
```
**3. Copiar a chave pública**
Mostra a chave pra você copiar:

```bash
ssh-add ~/.ssh/id_ed25519
```

Clonando um Repositório O comando git clone é usado para criar uma cópia idêntica de um repositório que já existe no GitHub (ou em outro servidor) dentro do seu computador.

O que o comando faz?
Diferente do git init (que começa um projeto do zero), o clone baixa todos os arquivos, todas as pastas e todo o histórico de mensagens (commits) de um projeto que já existe.

git clone URL_DO_REPOSITORIO
