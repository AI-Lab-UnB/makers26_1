 # Resumo Técnico sobre Git e Github

 Muitas das vezes, Git e GitHub são confundidos como sendo a mesma coisa, por este motivo, é essencial entender cada um deles como eles funcionam, além de suas diferenças.

 ## O que é Git

 O Git é um sistema de controle de versão de arquivos. É um software livre e muito utilizado no desenvolvimento de software onde diversas pessoas estão contribuindo simultaneamente, podendo criar e editar arquivos. 

 ## O que é o GitHub

 O GitHub é uma plataforma onde se pode armazenar seus projetos. É como se fosse uma rede social, só que com códigos, onde seus desenvolvedores podem disponibilizá-los para outras pessoas verem.

 ## Principais comandos Git

 | Comando | Função |
 |----------|---------|
 |git init | Inicializa um repositório|
 |git clone| Clona um repositório|
 |git status| Mostra o estado dos arquivos|
 |git add .| Adiciona arquivos para commit|
 |git commit -m "mensagem"| Salva alterações|
 |git push| Envia alterações para o GitHub|
 |git pull| Atualiza o repositório local|
 |git branch| Lista branches|
 |git checkout| Troca de branch|

 ## Fluxo básico de uso

1. Criar ou clonar um repositório
2. Fazer alterações nos arquivos
3. Executar git add
4. Executar git commit
5. Executar git push

## Conclusão

Git e GitHub são ferramentas essenciais para desenvolvimento de software, pois ajudam a organizar versões, facilitar o trabalho em equipe e manter o histórico das alterações.
