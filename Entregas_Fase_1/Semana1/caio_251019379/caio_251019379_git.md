# Resumo GIT / Github

---

# Workflow :

Ao trabalharmos com o GIT movemos os arquivos para alguns locais, sendo eles :

## Working Directory ( Diretório de Trabalho ) :

Aonde você na trabalha nos arquivos em sua máquina.

Cria arquivos, edita código, apaga, basicamente onde você constrói o projeto.

### git add ( prepara ) :

Comando para enviar o arquivo do working directory para stading area.

Ou seja, você está “selecionando” e deixando “em espera” os arquivos que futuramente serão commitados.

- **Exemplo :**

 `git add arquivo.text OU git add .`

 ---

## Stading Area ( Área de preparação ) :

Funciona quase como uma fila de arquivos, é o local onde você irá reservar os arquivos que deseja enviar para o local repo.

### git commit ( salvar versão ) :

Comando para enviar os arquivos que estão alocados na stading area para o local repo .

É uma boa prática que ao commitar algo também adicionemos uma mensagem explicando o que está contido nesse commit.

- **Exemplo :**

`git commit -m ‘meu primeiro commit’`

---

## Local Repository (Repositório Local) :

É o repositório que fica no seu computador de fato, funcionando como um histórico de commits, funciona offline, e geralmente é utilizado de forma individual/privada.

O repositório local é criado a partir do git init , ou a partir do git clone, o qual jávem na pasta .git.

### git push ( enviar ) :

Comando para enviar os commits armazenados no local repository para o remote repisitory.

Basicamente o comando para “uppar” seus arquivos para o github.

---

## Remote Repository  ( Repositório Remoto ) :

É o repositório em nuvem, on line, permite o compartilhamento de código entre uma equipe de maneira mais fácil, serve como backup 

### git pull ( baixar / atializar ) :

Comando para baixar ou atualizar o arquivo que esta no remote repository para sua máquina local, o working repository 

---

# Branch e Merge :

## Branch :

Um sistema essencial para o funcionamento do fluxo de trabalho utilizando git. 
Podemos gerar uma nova linha de produção alternativa à main, e utiliza-la para criar novas features, fazer testes e tratar bugs, sem interferir no código que já está testado e funcional na main.

### Comandos de branch :

- ***git checkout -b nome_da_branch*** ( ex : feature/login ) : cria uma nova branch e entra nela automaticamente
- ***git branch -d nome-da-branch : deleta a branch***
- ***git checkout nome_da_brench***, e o “ * “ sinaliza em qual você está.

---


## Merge :

É como juntamos as atualizações feitas nas branchs para a main.

Após testarmos e validarmos os códigos na branch vamos um push para a branch, depois criamos um “ pull request “ para o dono do repositório validar, e ele poderá dar merge, ou seja anexar os novos arquivos à main.

### Comandos de merge :

- ***git merge :***  realiza o merge pelo terminal

---

# Comandos "avulsos" :

- ***git status :*** mostra os arquivos presente nas branchs
- ***git log :*** mostra o histórico de commits
- ***git commit -am ‘mensage’ :*** adiciona e commita em apenas um comando
- ***git clone :*** gera uma cópia do repositório completo para sua máquina
- ***git fetch :*** baixa as ultimas mudanças, entretanto não altera seu arquivo
