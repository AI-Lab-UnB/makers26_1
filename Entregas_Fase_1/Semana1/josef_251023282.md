## GIT E GITHUB:
O Git é um **VCS (Version Control System)** distribuído e descentralizado. Ele permite que desenvolvedores gerenciem múltiplas versões de um projeto simultaneamente, facilitando a colaboração de uma equipe ou até mesmo organizar o projeto individualmente.

O **GitHub** é a plataforma que hospeda esses repositórios, oferecendo uma interface visual e ferramentas de trabalho.

---

## Inicializando o GIT:
Após instalar o Git, configure sua identidade no terminal (isso aparecerá nos seus commits):

```bash
git config --global user.name "Seu Nome"
git config --global user.email "seu.email@exemplo.com"
```
### Comandos principais do git:

```bash
git init  //para inicializar e criar uma pasta vazia .git
git add //para adicionar os arquivos no staging area
git commit -m  "(mensagem)"//adcionar um comentario do que foi feito
git push //manda para o repositorio
git pull //traz a versão mais nova do repositorio para sua maquina
git clone //pegar os arquivos de um novo repositorio (um projeto já começado por outro usuário) e copiar o projeto para o sua maquina
git remote “link do repositorio no github .git” //conectar o repositorio na sua maquina
git fetch //pega as ultimas mudanças sem dar merge
```
O Fork não é um comando do git, mas uma função do GitHub para você copiar o projeto de outra pessoa para a sua conta e poder mexer livremente.

## Arquivo .gitignore:

Serve para listar arquivos ou pastas que o Git deve ignorar. É essencial para não subir arquivos desnecessários ou sensíveis, como:

- Arquivos .env (onde ficam chaves de API e senhas)
- Pastas de dependências

## SSH Keys:

É uma forma de "dar permissão" para o seu computador acessar sua conta do GitHub sem precisar digitar senha toda hora. É como se fosse uma chave digital que o GitHub reconhece para saber que é você mesmo quem está enviando o código.

## Branch:
Permite criar uma linha separada da principal (main) e fazer alterações separadamente

- Comandos Básicos:
```bash
git branch //lista as branches existentes
git branch -b [nome-da-branch] //Cria e já entra na nova branch
git branch -d [nome-da-branch] //deleta a branch
git branch (nome-da-branch) //criar uma nova branch
```

## Pull Requests:
Quando você termina algo em uma branch separada, você abre um Pull Request. É basicamente um pedido para o dono do projeto (ou a equipe) revisar o que você fez. Se estiver tudo certo e sem conflitos, é feito o Merge (a união da sua branch com a principal).

## CI/CD (Integração Contínua/Entrega Contínua):
Ocorre um processo automatizado desde a escrita do código até a implantação. O código passa por testes rápidos e se tudo estiver certo, já é enviado direto para o site: deploy. Feito em sites de hospedagem como a Vercel. É necessário que logue com o github para manter a conexão com os commits novos feitos no repositório.
