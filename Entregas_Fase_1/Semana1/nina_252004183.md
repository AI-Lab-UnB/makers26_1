# O que é desenvolvimento profissional?

Desenvolvimento profissional não é apenas o acúmulo de habilidades técnicas ou a ascensão em cargos, mas um processo contínuo de construção de si mesmo no mundo do trabalho. É o encontro entre aquilo que sabemos fazer e aquilo que escolhemos nos tornar, moldado pela experiência, pelos erros e pelas reflexões ao longo do caminho.

Mais do que alcançar resultados externos, trata-se de um movimento interno de amadurecimento, em que o indivíduo aprende a lidar com desafios, a reconhecer seus limites e a expandir suas possibilidades.

Assim, desenvolver-se profissionalmente é, em essência, um exercício de transformação constante, no qual o trabalho deixa de ser apenas uma obrigação e passa a ser também um espaço de sentido, identidade e propósito.

# Resumo técnico — Git e GitHub

Este vídeo apresenta um guia técnico abrangente sobre Git e GitHub, focando desde conceitos fundamentais até fluxos de trabalho intermediários e automação.

Conforme as fontes, o Git é definido como um sistema de controle de versão distribuído (VCS), o que significa que cada desenvolvedor possui uma cópia completa do repositório em sua máquina local, permitindo o rastreamento de alterações, colaboração e reversão de versões.

Já o GitHub é a plataforma baseada na web que hospeda esses repositórios e oferece ferramentas de colaboração, como rastreamento de bugs e gerenciamento de tarefas.

## Principais tópicos abordados

### Configuração e Inicialização

- Configuração Inicial:  
Antes de começar, é necessário configurar a identidade do usuário com os comandos `git config --global user.name` e `git config --global user.email`, pois essas informações são vinculadas a cada commit. Também é recomendada a alteração do nome da ramificação padrão de "master" para "main" para alinhar com o padrão do GitHub.

- Inicialização: 
O comando `git init` cria um novo repositório local, gerando uma pasta oculta `.git` que armazena todo o histórico de alterações do projeto.

### Fluxo de Trabalho do Git (Workflow)

O fluxo básico consiste em mover arquivos entre quatro áreas principais:

1. Working Directory (Diretório de Trabalho):** Onde os arquivos são editados localmente.
2. Staging Area (Área de Preparação):** Onde os arquivos são marcados para inclusão no próximo commit através do comando `git add`.
3. Local Repository (Repositório Local):** Onde as alterações são gravadas permanentemente no histórico através do `git commit -m "mensagem"`.
4. Remote Repository (Repositório Remoto):** Versão hospedada (ex: GitHub) para a qual os commits são enviados com o `git push`.

### Colaboração e Ramificação (Branching)

- Branching: É uma funcionalidade essencial que permite criar linhas de desenvolvimento separadas (ex: `feature/login`) para trabalhar em novas funcionalidades ou correções sem afetar o código principal.
- Merging e Pull Requests: Após concluir o trabalho em uma branch, as alterações são integradas à branch principal (`main`) através de um merge local ou de um Pull Request (PR) no GitHub, que permite a revisão do código antes da fusão.
- Clonagem e Sincronização: O comando `git clone` é usado para copiar um repositório remoto inteiro para uma máquina local. Para manter o código atualizado com as mudanças de outros colaboradores, utiliza-se o `git pull`.

### Segurança e Boas Práticas

- .gitignore: Um arquivo crucial utilizado para listar arquivos e pastas que não devem ser rastreados pelo Git, como arquivos de configuração sensíveis (`.env`) contendo chaves de API.
- Autenticação SSH: As fontes detalham o uso de `ssh-keygen` para criar pares de chaves criptográficas, permitindo uma conexão segura e sem senha entre a máquina local e o GitHub.

### Automação com CI/CD

O vídeo demonstra a implementação de um pipeline de CI/CD (Integração Contínua e Implantação Contínua)utilizando a plataforma Vercel. Ao conectar um repositório do GitHub à Vercel, qualquer mudança enviada via `git push` para a branch principal dispara automaticamente uma nova implantação do projeto, tornando o processo de publicação rápido e eficiente.
