Problem Set 1:
Cálculos e Variáveis: Aprendi a manipular diferentes tipos de dados, como floats para valores monetários, e a importância da ordem de inicialização das variáveis para garantir a integridade dos cálculos. 
Estruturas de Repetição: Desenvolvi a lógica para simular o crescimento de economias mês a mês, integrando variáveis dinâmicas como aumentos salariais semestrais. 
Busca Binária: Aprendi que nem todo problema deve ser resolvido por força bruta. Implementar a busca binária para encontrar a taxa de juros ideal me ensinou a otimizar o tempo de processamento de um algoritmo.

Problem Set 2:
Criação de Funções: Aprendi a projetar funções auxiliares (como has_player_won e get_word_progress) para decompor um problema complexo — o jogo da Forca — em tarefas menores e gerenciáveis. 
Manipulação de Strings e Listas: Aprimorei a habilidade de iterar sobre strings e listas para comparar palpites do usuário com a palavra secreta, mantendo o estado do jogo atualizado. 

Problem Set 3
Dicionários em Python: Este foi o pset onde aprendi a utilizar dicionários como a estrutura de dados principal para mapear frequências de palavras, permitindo buscas e contagens muito mais rápidas do que com listas.
Métricas de Similaridade e TF-IDF: Aprendi a aplicar conceitos matemáticos e estatísticos na computação, calculando a similaridade entre documentos e utilizando a métrica TF-IDF (Frequência do Termo – Inverso da Frequência nos Documentos) para identificar quais palavras são realmente relevantes em um contexto.


Problem Set 4:
Recursão: Aprendi a utilizar a recursão como a ferramenta principal para navegar em estruturas não lineares, como árvores, calculando alturas e verificando propriedades de Heaps de forma elegante. 
Programação Orientada a Objetos (POO): Este pset consolidou meu entendimento sobre classes e herança. Aprendi a criar uma classe base Message para comportamentos comuns de criptografia e derivar classes específicas para lidar com textos claros ou cifrados. 
Criptografia e Decodificação: Desenvolvi a lógica para aplicar o método One-Time Pad (OTP) e, mais importante, criei algoritmos para quebrar códigos por meio de análise de frequência e busca em dicionários, unindo POO e lógica de processamento de texto.
