# Finger Exercise - 23

### Questão 1
**Premissa:** $n = a$

1. **Número de Iterações:** O loop `for i in range(5, a + 5)` executa exatamente `a` vezes.
2. **Custo por Iteração:** $O(1)$ (operações aritméticas e lógicas simples).
3. **Complexidade Total:** $a \times O(1) = O(a)$.
4. **Substituindo a variável:** Como $n = a$, a ordem de crescimento é linear.

**Resultado:** $\theta(n)$

### Questão 2
1. **Primeiro Loop:** `for e1 in L:`
   * O loop externo executa $n$ vezes.
   * A operação `if e1 in L2:` realiza uma busca linear em uma lista de tamanho $n$, custando $O(n)$ no pior caso.
   * Custo do primeiro bloco: $n \times n = O(n^2)$.

2. **Segundo Loop:** `for e2 in L2:`
   * O loop externo executa $n$ vezes.
   * A operação `if e2 in L:` também realiza uma busca linear em uma lista de tamanho $n$, custando $O(n)$ no pior caso.
   * Custo do segundo bloco: $n \times n = O(n^2)$.

3. **Complexidade Total:** $n^2 + n^2 = 2n^2$.
4. **Simplificação:** Desprezamos a constante 2.

**Resultado:** $\theta(n^2)$

### Questão 3
1. **Lógica do Loop:** `while n > 0:`
   * A cada iteração, o valor de $n$ é dividido por 10 (`n = int(n/10)`).
   * Isso significa que o loop executa uma vez para cada dígito do número $n$.

2. **Número de Iterações:** 
   * A quantidade de dígitos de um número $n$ é proporcional a $\log_{10}(n)$.

3. **Custo por Iteração:** 
   * Operações de módulo `%`, divisão `/` e soma `+` são consideradas de tempo constante, $O(1)$.

4. **Complexidade Total:** Proporcional ao número de vezes que podemos dividir $n$ por 10 até chegar a zero.

**Resultado:** $\theta(\log n)$
