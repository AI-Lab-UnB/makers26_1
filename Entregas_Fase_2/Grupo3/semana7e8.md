# Semana 7 – Planejamento do Projeto

## Objetivo da Semana

A Semana 7 foi dedicada à organização inicial do projeto **Onboarding Digital**, envolvendo a definição da proposta da aplicação, alinhamento da equipe e planejamento da arquitetura que será utilizada ao longo do desenvolvimento.

O foco principal foi estruturar as bases do projeto antes do início da implementação, garantindo que todos os integrantes compartilhassem a mesma visão sobre os objetivos, tecnologias e responsabilidades.

## Atividades Realizadas

### Formação da Equipe

Inicialmente foi realizada a organização do grupo de trabalho, definindo os integrantes responsáveis pelo desenvolvimento do projeto e estabelecendo uma divisão inicial de responsabilidades.

Essa etapa foi importante para facilitar a comunicação da equipe e distribuir as tarefas de forma mais equilibrada durante as próximas semanas.

Decidimos separa a equipe da seguinte forma: onde 2 integrantes ficarão responsáveis pelo FrontEnd, 2 pelo BackEnd, 1 DevOps e o Líder Técnico. A equipe optou por não rotacionar as duplas e se manter nas mesmas áreas de desenvolvimento até o final do projeto.

### Definição do Tema

Foi escolhido o tema **Onboarding Digital**, com a proposta de desenvolver uma solução voltada para auxiliar processos de integração e acompanhamento de usuários dentro de ambientes digitais.

A ideia central do projeto é oferecer uma experiência mais organizada e intuitiva para novos usuários, facilitando o acesso às informações e etapas necessárias durante o processo de onboarding.

### Planejamento da Arquitetura

Também foram realizadas discussões sobre a estrutura geral do sistema, definindo como os componentes da aplicação serão organizados.

Durante essa etapa foram analisadas questões relacionadas à separação entre frontend e backend, organização das funcionalidades e fluxo de comunicação entre as camadas da aplicação.

### Escolha da Stack Tecnológica

A equipe definiu as tecnologias que serão utilizadas durante o desenvolvimento do projeto.

| Camada | Tecnologia |
|------------|---------|
| FrontEnd | Next.js, TypeScript, Tailwind CSS |
| BackEnd | Python, FastAPI, SQLAlchemy |
| Banco de Dados | PostgreSQL |
| Infraestrutura | Docker, Docker Compose |
| Cloud | Google Cloud Platform |
| Documentação | GitHub Pages |

A definição prévia da stack permitiu reduzir dúvidas futuras e estabelecer um padrão de desenvolvimento para todos os integrantes.

### Organização do Repositório

Foi criado e configurado o repositório do projeto no GitHub, servindo como ambiente central para armazenamento do código-fonte, documentação e acompanhamento das atividades.

Também foram definidos padrões de organização de pastas, nomenclatura e fluxo inicial de trabalho utilizando branches e pull requests.

# Semana 8 – Estrutura Inicial do Projeto

Durante a Semana 8 foi realizada a configuração inicial da infraestrutura do projeto, preparando o ambiente de desenvolvimento para as próximas etapas de implementação.

## BackEnd
No backend, foi organizada a estrutura base da aplicação, incluindo a definição das pastas principais, configuração das dependências necessárias e preparação do ambiente utilizando Docker. Essa etapa permitiu estabelecer uma base sólida para o desenvolvimento das futuras funcionalidades e APIs do sistema.

## FrontEnd
No frontend, foi realizado o setup inicial utilizando **Next.js**, **TypeScript** e **Tailwind CSS**. A estrutura padrão do framework foi gerada, juntamente com os arquivos de configuração necessários para o desenvolvimento da interface da aplicação. Essa configuração fornece uma base moderna, escalável e de fácil manutenção para a construção das telas do sistema.

Além disso, foram organizados os arquivos de configuração do projeto e definidos os padrões iniciais de desenvolvimento, facilitando a integração entre os diferentes componentes da aplicação e o trabalho colaborativo da equipe.

Ao final da semana, o projeto já possuía a estrutura necessária para dar início ao desenvolvimento das funcionalidades planejadas nas próximas sprints.

## Conclusão Geral

As Semanas 7 e 8 foram essenciais para o início e a consolidação do projeto Onboarding Digital. Na Semana 7, o foco esteve no planejamento, permitindo definir os objetivos da aplicação, a organização da equipe, os papéis dos integrantes, a arquitetura do sistema e as tecnologias que serão utilizadas durante o desenvolvimento. Esse processo foi importante para alinhar as expectativas do grupo e estabelecer uma direção clara para o projeto.

Já na Semana 8, o trabalho concentrou-se na estruturação técnica da aplicação, com a configuração inicial dos ambientes de desenvolvimento e a criação da base do frontend e do backend. A adoção de tecnologias como Next.js, TypeScript e Tailwind CSS no frontend, juntamente com a organização inicial do backend, proporcionou uma fundação sólida para as próximas etapas de implementação.

Ao final dessas duas semanas, o projeto passou a contar tanto com um planejamento bem definido quanto com uma estrutura técnica preparada para receber as funcionalidades previstas, permitindo que a equipe avance para as próximas fases de desenvolvimento de forma organizada e consistente.
